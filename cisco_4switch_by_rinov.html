Panduan Cisco Packet Tracer — Router 2911 + 4 Switches

Langkah-demi-langkah lengkap (IP, konfigurasi router, tabel IP PC, dan perintah CLI). Siap di-*copy paste* ke GitHub.

Ringkasan Tujuan

Membagi 20 PC menjadi 4 jaringan (5 PC per switch). Router 2911 bertindak sebagai gateway untuk tiap jaringan (tidak menggunakan VLAN trunking — koneksi langsung interface router ke switch). Subnet yang dipakai bersifat /24 untuk kejelasan.

Langkah 1 — Membangun Topologi di Packet Tracer

  1. Tambahkan Router 2911 dan 4 switch (mis: 2960).
  2. Tambahkan 20 PC, bagi 5 per switch.
  3. Hubungkan Router G0/0 → Switch1, G0/1 → Switch2, G0/2 → Switch3, G0/3 → Switch4 (kabel straight-through).
  4. Hubungkan setiap PC ke port switch menggunakan kabel straight-through.

Langkah 2 — Atur IP pada tiap PC

Klik PC → Desktop → IP Configuration. Masukkan IP dan Default Gateway sesuai tabel di atas (Subnet mask 255.255.255.0).

Contoh konfigurasi PC (Switch 1, PC1)
IP Address: 192.168.1.2
Subnet Mask: 255.255.255.0
Default Gateway: 192.168.1.1
DNS: (kosong atau 8.8.8.8)

Langkah 3 — Konfigurasi Router (CLI)

Buka Router → CLI dan masukkan perintah berikut (satu per satu). Interface yang dipakai: GigabitEthernet0/0 sampai 0/3.

Perintah CLI Router (seluruh interface)
enable
configure terminal

! Interface ke Switch 1
interface gigabitEthernet0/0
 ip address 192.168.1.1 255.255.255.0
 no shutdown
 exit

! Interface ke Switch 2
interface gigabitEthernet0/1
 ip address 192.168.2.1 255.255.255.0
 no shutdown
 exit

! Interface ke Switch 3
interface gigabitEthernet0/2
 ip address 192.168.3.1 255.255.255.0
 no shutdown
 exit

! Interface ke Switch 4
interface gigabitEthernet0/3
 ip address 192.168.4.1 255.255.255.0
 no shutdown
 exit

end
write

Catatan: Jika router modelmu tidak punya gigabit 4 port, gunakan modul atau sesuaikan port (misal serial/fastEthernet) — ubah nama interface dalam perintah di atas).

Langkah 4 — Uji Koneksi

Dari PC1: buka Command Prompt → ping 192.168.2.2 (PC pada jaringan lain). Jika menerima reply, konfigurasi antar-jaringan berhasil.

Contoh perintah ping dari PC (Desktop → Command Prompt)
ping 192.168.2.2
ping 192.168.3.2
ping 192.168.4.2

Tips Troubleshooting Singkat

  • Pastikan interface router no shutdown.
  • Periksa kabel tipe straight-through (PC→Switch dan Router→Switch juga straight di PT).
  • Pastikan Gateway PC sesuai dengan IP interface router di jaringan yang sama.
  • Gunakan show ip interface brief di router untuk melihat status interface.
  • Jika tidak ada reply, coba ping gateway dari PC untuk memastikan koneksi fisik.
